## Decisions and reasoning

- **Token architecture:** single always-on token set in `:root` (DESIGN_SYSTEM explicitly sanctions "an equivalent single always-on token set"); scaffold light `:root` block AND achromatic `.dark` block both deleted. A permanent `dark` class on `<html>` (layout.tsx) keeps the vendored primitives' `dark:`-variant tunings active (button outline/ghost hovers, destructive tints, aria-invalid rings). Chose the conventional class-marker over redefining `@custom-variant dark (&)` as unconditional — same effect, zero cleverness risk, future CLI-added primitives work untouched. Constraint comments left in globals.css and layout.tsx (the permanence is not inferable from code).
- **Supporting tokens** (values DESIGN_SYSTEM's tables don't name — declared, derived from documented principles, recorded in a new DESIGN_SYSTEM Supporting Tokens table): `--secondary` 10% off-white (the "white-at-low-opacity fill" secondary-button wording), `--muted` 8%, `--accent` 10% + full-tier foreground (hover/highlight — never teal; teal = active/live only), `--border` 10% hairline, `--input` 12% (border + disabled fill; the fill proper is `--well`), `--ring` = `var(--primary)` (focus = interactive = teal), sidebar family as 1:1 aliases (nothing consumes them until Wave 4).
- **Aliases via `var()`** (`--positive`, `--ring`, `--card-foreground`, sidebar family) rather than repeated literals — captures DESIGN_SYSTEM's own "same as" language and prevents drift.
- **`--chart-3..5` left as scaffold grays** — DESIGN_SYSTEM explicitly rules them open; inert until Wave 5. Comment records this.
- **NOT done, deliberately:** no light-mode anything (grep-verified none exists: no prefers-color-scheme/ThemeProvider/next-themes in src); no logic/data-layer changes (none needed — nothing rendered wrong for a logic reason); Card's `ring-1 ring-foreground/10` hairline kept (the "minimal" in no/minimal strokes); shell/header `border-b`/`border-l`/`border-dashed` hairlines kept (same rule); 03a build file untouched (the route+shell item's FLAGGED text is accurate history; no checklist item covers this supplemental fold).
