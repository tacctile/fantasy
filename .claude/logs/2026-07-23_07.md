## Clarify (AskUserQuestion — 2 questions, Nick-signed)

## Build (3 edits + 1 new file, one fold)

1. **New `components/dashboard/share-link-panel.tsx`** — `'use client'` `ShareLinkPanel({ leagueId, shareToken })`. Card (header explains the read-only, no-login spectator view). Readonly Input showing `<origin>/share/<token>` (origin via `useOrigin()` — a `useSyncExternalStore` read: `''` on server/first client render, `window.location.origin` after — the draft-queue store's hydration-safe pattern, chosen after eslint `react-hooks/set-state-in-effect` rejected the initial useEffect+setState approach). Copy button (navigator.clipboard, Check feedback + sonner toast). Two-step regenerate: default outline button → confirming state (warning panel using --warning tokens) with destructive Confirm (RefreshCw spin while pending) + ghost Cancel; Confirm calls `regenerateShareToken(leagueId)`, updates local `token` from the returned value, toasts per outcome (regenerated/unauthorized/not_found + catch).
2. **`league-dashboard-shell.tsx`** — new optional `settingsSlot?: React.ReactNode`, rendered as a `max-w-2xl` `<section>` at the end of `<main>` after the standings/power grid. A layout slot — the shell stays agnostic about its contents, keeping the admin-settings concern out of the pure data composition.
3. **`dashboard/page.tsx`** — new page-local `fetchShareToken(db, leagueId)` (explicit `select('share_token').eq('platform_league_uuid', leagueId).maybeSingle()` via the RLS server client — owner policy grants it; deliberately NOT in dashboard.ts). Added to the initial `Promise.all`; passes `<ShareLinkPanel>` as `settingsSlot` (or `undefined` when the token is somehow null — the panel simply doesn't render, no crash; the layout already 404s unknown leagues so this is defensive only).

Disclosed judgment calls: (a) `share_token` read lives in the page, not a service — the one owner-only surface managing the link, kept physically contained rather than widening dashboard.ts's boundary or spreading the token through ConnectedLeague; (b) display driven by local `token` state (init from prop, updated on regenerate's returned value) — the panel is the only mutator, so local state is the source of truth post-regenerate even before revalidatePath lands; (c) copy reads `window.location.origin` fresh at click time (robust regardless of the render-time origin snapshot).
