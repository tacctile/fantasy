## Build narrative

- `listConnectedLeagues()` + `ConnectedLeague` added to `src/services/draft-board.ts` (explicit columns; boundary fields never selected; deterministic order).
- `src/lib/supabase/auth.ts`: `getAdminAuthState()` — `getUser()` then `rpc('is_fantasy_admin')` (typed since the types regen); three-state result (`unauthenticated`/`not_admin`/`admin` + email).
- `src/proxy.ts` (Next 16 convention): Supabase session refresh on every matched request (RSC can't write cookies; without this, refresh-token rotation eventually signs the owner out); unauth `/leagues/*` → 307 `/login` with rotated cookies carried onto the redirect; matcher excludes `/api/`, `_next/*`, favicon.
- `/login`: RSC page (authed sessions bounce to `/`), client `LoginForm` (`useActionState`), server actions `signIn` (generic error only — provider internals never echoed; success redirects to landing) and `signOut`. Landing = first connected league's board; zero leagues (or RLS-empty non-admin list) → `/`.
- `(admin)/layout.tsx` gate: unauthenticated → `/login`; `not_admin` → `NotAuthorizedCard` (zero league data, sign-out only) — children never render. Third wall = RLS.

- Root `page.tsx` rewritten from Wave 1 placeholder to auth-aware router (unauth → login; non-admin → not-authorized; admin → first board; zero leagues → minimal note).
