## Decisions and implementation notes

- **Item 4 — `src/services/bpa/board.ts` `computeBpaBoard(pool, layout, leagueSize, adp)`:** THE one composition of replacement → calibration → base value; pure, no cache. The shrinking pool is realized by the CALLER passing the current undrafted set (drafted removed) — so replacement is always the last startable among those still available. **Disclosed recompute model (Nick-signed via the pure layer + STATE self-location):** full league-wide demand re-run against the shrinking pool — it does NOT subtract already-filled league starter slots (that would need per-team remaining-slot tracking). Negligible with the ~3.3k projection pool (never exhausts in the real decision window, picks 1–100); reducing demand by filled slots is a possible future refinement, noted, not a v1 requirement. The build file's item-4 text ("recompute as the pool shrinks") is satisfied literally.
- **Item 5 — `src/services/bpa/need.ts` `computeRosterNeed`/`classifyPositionNeed`:** pure per-team greedy fill (dedicated then flex, narrowest-eligibility-first, matching replacement.ts's order). Reuses `FLEX_ELIGIBILITY`, now **exported from replacement.ts** as the single flex-eligibility home (need.ts never re-derives). Bench modeled GENERIC (one roster-wide `benchSlotsRemaining`, not per-position — wiki-grounded). Need kinds starter/bench/full per Q3. Flex-slot consumption order among eligible positions is provably immaterial to every output (documented). Structurally its own module → item 6 attaches it as a parallel annotation, never a value input.
- **Item 6 — `src/services/bpa/recommendations.ts` `getBpaRecommendations` (first I/O in bpa/) + `fetchBpaRecommendations` action (draft/actions.ts):** loads config (weights via `parseScoringWeights`, layout via `parseRosterSlotLayout`, format via `resolveAdpScoringFormat`, size from derived_config), **paginated projections** (a `.range` loop, 1000/page — the 3292-row season snapshot exceeds PostgREST's default page cap; a single unbounded select would silently truncate), format ADP carriers, and `listDraftPicks` ONCE (→ drafted set for the shrinking pool + self-roster positions for need — reusing the catalog-joined `playerPosition` the snapshot already carries). Runs `computeBpaBoard` on the shrunk pool, slices top-N (default 10), annotates each row with `classifyPositionNeed`. `selfRosterId` optional (Q1). Typed reasons `league_not_found`/`config_unusable`/`projections_unavailable`; **ADP absence is not a failure** (calibration degrades to structural_only). ADP carriers passed to calibration are NOT drafted-filtered — depth is the market's stable structural preference (calibration.ts's signed design: "market sets depth, projections set value"); the VALUE deepens via the shrinking projection pool. Action = same in-action `getAdminAuthState` + RLS-server-client gate as the other 5 actions; a read, so no revalidate.
