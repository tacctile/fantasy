## Item 3 — roster-construction-aware auto-pick → [x]

`positionCap(pos, layout)` = dedicated starters + flex slots the position is eligible for (reads `FLEX_ELIGIBILITY` from THE one home, replacement.ts) + a per-position bench margin (`AUTO_PICK_BENCH_MARGIN`: QB 1 / RB 4 / WR 4 / TE 2 / K 0 / DEF 0, default 3 — DECLARED WIKI SILENCE, live-tuned). K/DEF cap = 1 (never a 2nd); QB in 1QB = 2 (the build-file 3rd-QB example); superflex lifts QB for free (SUPER_FLEX flex eligibility). `selectAutoPickCandidate` walks draftable queue entries, skips any at cap, returns the first under cap; if all are capped, falls through to the top draftable one (`overCapFallThrough`, never fail to pick); null position / null layout → take the top (honest degradation, no guessed cap). Trigger (shell): an armed toggle (off by default) fires when `projectOnClock(nextPickNumber, draftOrder) === selfRoster` AND away — hidden tab fires immediately, else a 12s dwell (`AUTO_PICK_DWELL_MS`, local proxy for the clock winding down) with a visibilitychange→hidden firing early. `autoPickAttemptRef` = one attempt per pick number (no retry loop). Fires through the shell's SAME `handleDraft` → `recordManualPick` (source='manual', undoable, first-write-wins: a poller-won pick makes the attempt conflict + roll back, so it can NEVER desync a real Sleeper draft). NEVER a Sleeper write.
